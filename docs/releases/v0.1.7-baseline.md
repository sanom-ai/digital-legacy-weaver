# v0.1.7 Baseline (Core Flow + Security/Keys + Documentation)

This document locks the `v0.1.7` baseline for controlled rollout and team onboarding.

## Baseline Objective

1. Keep core flow stable for day-to-day testing:
- owner setup -> trigger path -> recipient unlock flow
2. Keep security controls and key hygiene operational:
- runtime dispatch guard
- release preflight and safety drills
- explicit key rotation policy
3. Keep documentation complete and actionable:
- production deploy runbook
- runtime secrets setup
- incident response and release checklist

## Migration Baseline Rule

To avoid drift and `db push` conflicts:

1. Migration filename must use unique 14-digit version prefix:
- Example: `20260405000100_init.sql`
2. Keep one migration file per version.
3. Never reuse shortened date-only versions (`20260405`, `20260406`, etc.).

## One-command Baseline Gate

Run this before rollout decisions:

```powershell
python tools/v017_baseline_gate.py --max-age-days 30
```

Optional strict mode (requires clean working tree):

```powershell
python tools/v017_baseline_gate.py --max-age-days 30 --require-clean-tree
```

## Decision Policy

1. `PASS` only:
- move forward with controlled rollout / partner demo / beta batch
2. Any `FAIL`:
- fix failed check first, then rerun baseline gate
