# v0.1.3 - Human-First Product Polish 3

Release date: April 7, 2026
Type: Product polish and trust hardening release

## TL;DR
This release turns core flows into a more human-first experience: cleaner onboarding, unified UI language across dashboard and builder, safer and clearer beneficiary receipt flow, and consistent visual behavior in settings/connectors. Security behavior remains strict, while readability and confidence are significantly improved.

## User outcome (before -> after)
- Before: First-time users saw mixed visual patterns and higher cognitive load across setup pages.
- After: Setup is guided with clearer hierarchy, more consistent input styles, and stronger step clarity.

- Before: Dashboard and Intent Builder looked like separate tools.
- After: Both now share one Material 3 visual language for cards, status blocks, chips, and actions.

- Before: Beneficiary flow had strong security controls but felt denser to read.
- After: Beneficiary unlock/receipt flow is easier to follow, with safer form presentation and clearer anti-scam cues.

- Before: Safety settings and connectors had partial visual drift.
- After: Sections are now visually aligned with the rest of the app, improving product coherence.

## What’s included
### 1) Onboarding completion polish
- Improved step intro readability and spacing.
- Unified input style with consistent borders, fills, icon cues, and helper placement.
- Better scanability for contacts, triggers, and consent sections.

### 2) Dashboard consistency pass
- Reworked dashboard action cards to a unified style and hierarchy.
- Reduced hard-coded surface colors; adopted theme-driven surfaces and borders.
- Refined status chips and helper panels for a cleaner “product” feel.

### 3) Intent Builder consistency pass
- Polished top sections (plan intro, presets, partner area, ecosystem area).
- Improved partner card readability and selected-state clarity.
- Unified status banner and chip styles with app-wide theme behavior.

### 4) Beneficiary receipt flow polish (no security logic change)
- Upgraded unlock form field presentation for clarity and confidence.
- Improved journey/status panel visuals and message banner readability.
- Kept anti-scam flow and lock/cooldown behavior unchanged.

### 5) Safety settings + connectors polish
- Aligned section cards, inputs, and feedback panels with Material 3 language.
- Improved dialog form readability for destination path and asset reference management.

### 6) Global theme refinement
- Strengthened typography/spacing consistency at theme level.
- Added and tuned dialog/list/button/input/expansion tile behavior for coherent UX.
- Improved tap targets and control consistency for day-to-day usage.

## Trust and security posture
- Private-first model remains intact.
- App does not store actual balances or transfer amounts.
- Sensitive money-like text stays redacted where export/reference rules apply.
- Recipient flow still enforces anti-scam confirmation and temporary lock controls.
- No downgrade of security gates in this release.
