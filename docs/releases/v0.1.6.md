# v0.1.6 - Release Pipeline Reliability + Store Pack Confirmation

Release date: April 8, 2026  
Type: Release operations hardening

## TL;DR

This release confirms a stable, repeatable app release pipeline with automatic Android and Windows asset publishing, aligned production runtime secrets, and updated deploy/runbook guidance for operators.

## User outcome (before -> after)

- Before: App builds could succeed but release publishing could fail on path edge cases.
- After: Release workflow now handles artifact path normalization safely and completes publish reliably.

- Before: Runtime secret setup depended on scattered knowledge.
- After: Runtime secret source and setup steps are documented in one production guide.

- Before: Teams needed manual fallback steps more often to attach Android assets.
- After: App Release Pack now publishes `AAB/APK/Windows ZIP` automatically on successful run.

## What changed

1. Release workflow hardening
- Added artifact normalization step before publish.
- Added idempotent same-file guard for normalize copy.
- Kept fail-fast behavior for truly missing assets.

2. Production ops documentation
- Added `docs/runtime-secrets-setup.md` as runtime secret source-of-truth.
- Linked runtime secret guide from production deploy runbook.
- Updated app release pack notes to reflect normalized publish flow.

3. Runtime environment alignment
- Production Supabase runtime secrets synchronized for dispatch/reviewer/handoff flow.
- GitHub Actions secret names aligned with current production runbook.
