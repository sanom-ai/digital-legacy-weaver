# v0.2 Owner Assignment Matrix

This matrix defines clear accountability for execution speed, safety, and release quality.

## Role Keys
- `PO` = Product Owner
- `SEC` = Security Lead
- `DPO` = Data Protection / Privacy Owner
- `ENG` = Engineering Lead
- `OPS` = Operations / Release Owner
- `LEGAL` = Legal Counsel / Legal Owner
- `PARTNER` = Partner Network Owner
- `QA` = Quality Assurance Owner

## RACI Guide
- `R` Responsible (does the work)
- `A` Accountable (final owner)
- `C` Consulted (provides input)
- `I` Informed (kept updated)

## Escalation Rules
- Any `P0` security blocker: escalate to `SEC + ENG + OPS` immediately.
- Any legal ambiguity affecting release: escalate to `LEGAL + DPO + PO`.
- Any partner fee transparency gap: block partner listing until `PARTNER + LEGAL` sign-off.
- Any failing release gate: `OPS` blocks release until fixed or risk-approved by `PO + SEC`.

## Decision Cadence
- Daily: execution sync using kanban board.
- Weekly: risk and compliance review across `PO/SEC/DPO/OPS`.
- Release checkpoint: final sign-off from `OPS (gate)`, `SEC (security)`, `PO (user outcome)`.
