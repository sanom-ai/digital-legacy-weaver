## Recommended Usage Flow
1. Use the Execution Board to define scope and exit criteria.
2. Run daily delivery using the Kanban Board.
3. Keep accountability clear with the Owner Assignment Matrix.
4. Gate every release decision with the Go/No-Go Scorecard.

## Governance Rhythm
- Daily: Update Kanban statuses and blockers.
- Weekly: Review cross-functional risk (product, security, legal, ops).
- Release checkpoint: Complete Go/No-Go and collect sign-offs.

## Single-Page Health Check
- Are all P0 items either `done` or explicitly risk-accepted?
- Is security posture still metadata-only backend + zero-knowledge boundary target?
- Are legal/compliance docs current and linked to current release scope?
- Are quality gates passing (`analyze`, `test`, preflight, security triage)?
