# v0.2 Execution Board (No-Date Fast Track)

## Operating Rules
- No sensitive payload in backend storage.
- Every risky action must be audit-logged.
- No release without quality + security gates passing.
- Human-first UX copy must remain clear for non-technical users.

- `in_progress` = active work
- `blocked` = waiting dependency

- `done` = complete and verified
