# v0.2 Go/No-Go Release Scorecard

Use this scorecard at final release checkpoint.  
Decision must be explicit: `GO` or `NO-GO`.

## Decision Rule
- `GO` only if all `P0` checks are pass, and no unresolved critical security/compliance blocker exists.
- `NO-GO` if any of the following is true:
  - Any `P0` gate fails
  - Any unresolved `Critical` or `High` security finding without formal risk acceptance
  - Any legal/compliance blocker marked unresolved

## Scoring
- Each section has required checks.
- Mark each check: `PASS`, `FAIL`, or `RISK-ACCEPTED`.
- Summary:
  - `PASS`: no action needed
  - `FAIL`: must fix before release
  - `RISK-ACCEPTED`: must include approver name and reason
